# Railway Deployment Variables

This document specifies all required environment variables for deploying the FrogmenDash ERP application to Railway.

## Required Variables (App Service)

Configure these in your Railway App Service environment:

| Variable | Value | Description |
|----------|-------|-------------|
| `APP_NAME` | `FrogmenDash` | Application name |
| `APP_ENV` | `production` | Environment mode |
| `APP_DEBUG` | `false` | Debug mode (disable in production) |
| `LOG_LEVEL` | `info` | Logging level |
| `APP_KEY` | `base64:...` | Laravel encryption key (generated by `php artisan key:generate`) |
| `DB_CONNECTION` | `mysql` | Database driver |
| `DB_HOST` | `caboose.proxy.rlwy.net:49972` | MySQL TCP proxy host (from Railway MySQL service) |
| `DB_PORT` | `49972` | MySQL port |
| `DB_DATABASE` | (your database name) | Database name |
| `DB_USERNAME` | (your MySQL username) | MySQL username |
| `DB_PASSWORD` | (your MySQL password) | MySQL password |
| `SESSION_DRIVER` | `file` | Session storage (file = default, database = after first boot) |
| `CACHE_DRIVER` | `file` | Cache storage (file = default, database = after first boot) |
| `APP_ADMIN_EMAIL` | `admin@example.com` | First admin user email (only for fresh installs) |
| `APP_ADMIN_PASSWORD` | (secure password) | First admin user password (only for fresh installs) |

## Notes

### APP_URL
- **Automatically set by bootstrap script** from `RAILWAY_PUBLIC_DOMAIN` environment variable
- You can optionally set it manually: `https://your-domain.up.railway.app`
- If not set, Railway's automatic domain (`your-app-production.up.railway.app`) will be used

### RAILWAY_PUBLIC_DOMAIN
- Automatically provided by Railway platform
- App service receives this automatically; no manual configuration needed

### MySQL Connection
- Use TCP proxy (`caboose.proxy.rlwy.net`) instead of private DNS
- Port varies; check your Railway MySQL service details
- Extract from Railway MySQL service: "TCP Proxy URL" field

### First Deploy vs. Redeploy
- **First deploy (empty DB):** Boot script auto-runs installer, creates admin user
- **Redeploying:** Boot script only runs migrations; existing data preserved
- To force reinstall: Delete all data and set `APP_ADMIN_EMAIL` + `APP_ADMIN_PASSWORD`

### Security
After successful first deployment:
1. **Rotate MySQL password** (exposed in app variables)
2. **Change admin password** (default `12345678` used during install)
3. **Rotate APP_KEY** if needed (invalidates encrypted sessions/data)

### Session & Cache Drivers
- **Production:** Switch to `database` after confirming stable deployment
- **First boot:** Uses `file` to avoid table-existence issues during migrations
- To switch: Update variable, redeploy

## Troubleshooting

### Mixed Content Errors (CSS/JS not loading)
- **Cause:** APP_URL not set to HTTPS or proxy headers not trusted
- **Solution:** Already configured in bootstrap.php and app.php; ensure RAILWAY_PUBLIC_DOMAIN is set
- **Check:** Browser console shows 'https://' for page but 'http://' for resources

### Admin User Creation Fails
- **Cause:** Company not created by seeder
- **Solution:** Bootstrap script now auto-creates default company if missing
- **Check:** Deploy logs show "🏢 Creating default company..." message

### Database Connection Fails
- **Cause:** Host/port/credentials incorrect
- **Solution:** Extract from Railway MySQL service "TCP Proxy URL"
- **Format:** `caboose.proxy.rlwy.net:PORT` (use this exact format)

### Migrations Fail
- **Cause:** Database username lacks privileges
- **Solution:** Ensure DB user has full permissions on target database
- **Check:** Deploy logs show migration names during boot

## Railway Quick Start

1. Create MySQL service (Railway platform)
2. Create App service (GitHub repo)
3. Connect app to MySQL via "Public Networking" or "Private Networking"
4. Extract MySQL connection details, set variables above
5. Deploy and monitor logs for success
6. Login with admin credentials, change password
